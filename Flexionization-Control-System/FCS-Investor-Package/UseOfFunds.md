# Use of Funds

This document outlines the allocation of investment required to develop the FCS project over a 12-month period.  
Target raise: **$300,000 – $500,000** (pre-seed).

---

## 1. Summary Allocation

| Category                         | Purpose                                           | Allocation |
|----------------------------------|---------------------------------------------------|------------|
| **Engineering & R&D**           | Core team development, math, integration          | **40%**    |
| **Hardware & Testing**          | UAV/robotics hardware, sensors, test benches      | **30%**    |
| **Developer Ecosystem & Docs**  | Documentation, SDK tools, community               | **20%**    |
| **Partnerships & Integrations** | Pilot programs, integrations, travel              | **10%**    |

---

## 2. Detailed Breakdown

### **2.1 Engineering & R&D — 40%**
Funding required for:
- 2–3 embedded C++ engineers  
- UAV/robotics integration specialist  
- applied math support (FXI–Δ–E model improvements)  
- SDK feature expansion and optimization  
- real-time performance improvements  

**Estimated cost:** $120,000 – $200,000

---

### **2.2 Hardware, Prototyping & Testing — 30%**
Includes:
- UAV platforms  
- robotic arms / servos  
- IMU, barometer, LiDAR sensors  
- STM32 / PX4 / ESP32 controllers  
- creating a physical test bench  
- field testing equipment  

**Estimated cost:** $90,000 – $150,000

---

### **2.3 Documentation, Developer Tools & Community — 20%**
Covers:
- extended documentation  
- tutorials and integration guides  
- developer tooling  
- CI/CD and repository maintenance  
- open-source ecosystem growth  
- community engagement  

**Estimated cost:** $60,000 – $80,000

---

### **2.4 Partnerships & Integrations — 10%**
Funds allocated for:
- industry collaborations  
- pilot deployments with UAV/robotics companies  
- travel for integration and testing  
- certification-related consultations  

**Estimated cost:** $30,000 – $50,000

---

## 3. Timeline

Funding covers **12 months** across four phases:

1. **Q1** — SDK stabilization, documentation, initial demos  
2. **Q2** — PX4/ROS2 integration, hardware test bench creation  
3. **Q3** — advanced modules, auto-tuning, MCU packages  
4. **Q4** — commercial readiness, pilot integrations  

---

## 4. Goal

This investment will enable FCS to:

- build the core engineering team  
- complete and validate hardware integrations  
- release **FCS v1.0** for commercial adoption  
- prepare for partnerships and industry scaling
